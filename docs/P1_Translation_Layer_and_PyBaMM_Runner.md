# P1 — Translation Layer + PyBaMM Runner

## Context

FORGE is a parametric battery cell design and simulation platform. The ML autoresearch system (Layer 3) has been validated using synthetic data generated by proxy formulas. This prompt implements the **real physics pipeline**: a translation layer that converts geometric cell design parameters into PyBaMM simulation inputs, and a PyBaMM DFN simulation runner that produces actual electrochemical/thermal performance data.

The translation layer **must reuse existing Layer 1 winding calculations** — this demonstrates L1→L3 integration and avoids reimplementing validated geometry code.

## What to Build

### File 1: `forge/ml/simulation/geometry_translator.py`

**Purpose:** Translate 8 geometric design parameters into a dict of PyBaMM parameter overrides that can be applied to a base parameter set (Chen2020).

**Class: `GeometryTranslator`**

**Input parameters (the 8 geometric features):**

| Parameter | Range | Unit | Type |
|-----------|-------|------|------|
| `electrode_thickness` | 50–150 | μm | float |
| `porosity` | 0.20–0.50 | — | float |
| `separator_thickness` | 10–50 | μm | float |
| `n_tabs` | 1–6 | count | int |
| `tab_width` | 5–30 | mm | float |
| `can_inner_diameter` | 44–46 | mm | float |
| `can_wall_thickness` | 0.2–0.8 | mm | float |
| `cell_height` | 65–95 | mm | float |

**Method: `translate(params: dict[str, float]) -> dict[str, float]`**

Returns a flat dict of PyBaMM parameter name → value overrides. The mapping logic:

#### Direct mappings (electrode/separator)

- `electrode_thickness` (μm) → `"Positive electrode thickness [m]"` = value × 1e-6
- `electrode_thickness` (μm) → `"Negative electrode thickness [m]"` = value × 1e-6 × 1.1 (standard N/P ratio — anode 10% thicker than cathode for safety margin)
- `porosity` → `"Positive electrode porosity"` = value
- `porosity` → `"Negative electrode porosity"` = value (simplification: both electrodes calendared to same porosity target)
- `separator_thickness` (μm) → `"Separator thickness [m]"` = value × 1e-6

#### Contact resistance from tab geometry

`n_tabs` + `tab_width` → `"Contact resistance [Ohm]"`

Physics: Each tab provides a parallel current path. More/wider tabs reduce effective resistance.

Formula:
```python
# Reference values (from Chen2020 baseline — a single-tab cell)
R_CONTACT_BASELINE = 0.01  # Ohm — Chen2020 default contact resistance
N_TABS_BASELINE = 1
TAB_WIDTH_BASELINE = 10.0  # mm — reference tab width

# Tab conductance scales linearly with total tab cross-section
tab_conductance_ratio = (n_tabs * tab_width) / (N_TABS_BASELINE * TAB_WIDTH_BASELINE)
R_contact = R_CONTACT_BASELINE / tab_conductance_ratio
```

This produces `"Contact resistance [Ohm]"` in the override dict.

#### Jellyroll geometry from can dimensions (REUSE L1)

`can_inner_diameter`, `can_wall_thickness`, `cell_height` → electrode area, electrode height, and thermal parameters.

**Step 1 — Compute jellyroll radii:**
```python
can_outer_radius = can_inner_diameter / 2  # mm (note: can_inner_diameter IS the inner diameter of the can)
jellyroll_outer_radius = can_outer_radius - 0.15  # mm, ~0.15mm radial clearance (sliding fit)
mandrel_radius = 2.5  # mm for 4680-class cells (fixed, standard)
```

**Step 2 — Compute layer thickness per wind:**
```python
# One full wind = anode + cathode + 2×separator + 2×current collector
cc_al_thickness = 16e-3  # mm (aluminum, cathode side) — from Chen2020
cc_cu_thickness = 12e-3  # mm (copper, anode side) — from Chen2020
separator_mm = separator_thickness / 1000  # convert μm to mm
pos_electrode_mm = electrode_thickness / 1000
neg_electrode_mm = pos_electrode_mm * 1.1  # N/P ratio

layer_thickness_per_wind = (
    pos_electrode_mm * 2  # cathode double-coated
    + neg_electrode_mm * 2  # anode double-coated
    + separator_mm * 2  # separator on each side
    + cc_al_thickness  # aluminum collector
    + cc_cu_thickness  # copper collector
)
```

**Step 3 — Call L1 winding calculations:**

Import and use the functions from `forge/engine/calculations/winding.py` to compute:
- Number of winds (from `mandrel_radius`, `jellyroll_outer_radius`, `layer_thickness_per_wind`)
- Electrode length (from spiral geometry)
- Electrode area = electrode_length × electrode_height_m (both sides)

For `electrode_height`:
```python
# Cell height minus top/bottom clearance (header, vent, insulation)
HEADER_CLEARANCE_MM = 5.0  # top: PTC/CID/vent/cap assembly
BOTTOM_CLEARANCE_MM = 3.0  # bottom: insulator disc + weld
electrode_height_mm = cell_height - HEADER_CLEARANCE_MM - BOTTOM_CLEARANCE_MM
```

**IMPORTANT:** Read `winding.py` carefully before calling it. Understand what units it expects (mm vs m), what parameters the functions take, and what the return types are. Adapt the calling code to match — do NOT reimplement the winding math.

The resulting values map to PyBaMM parameters:
- `"Electrode height [m]"` = electrode_height_mm × 1e-3
- `"Electrode width [m]"` = electrode_length_from_winding × 1e-3 (PyBaMM calls the unwound electrode length "width" in its 1D model)
- `"Positive electrode thickness [m]"` (already set above)
- `"Negative electrode thickness [m]"` (already set above)
- `"Number of electrodes connected in parallel to make a cell"` = 1 (wound cell, single jellyroll)

#### Thermal parameters from can geometry

For the lumped thermal model, PyBaMM needs cell-level thermal properties:

```python
# Cell volume and surface area (for lumped thermal)
r_outer = (can_inner_diameter / 2 + can_wall_thickness) * 1e-3  # m, outer radius including wall
h_total = cell_height * 1e-3  # m

cell_volume = math.pi * r_outer**2 * h_total  # m³
cell_surface_area = 2 * math.pi * r_outer * h_total + 2 * math.pi * r_outer**2  # m² (cylinder)

# Total cell mass estimate (jellyroll + can)
# Can mass from hollow cylinder
can_density = 7900  # kg/m³ (steel, standard 4680 can material)
r_inner_can = (can_inner_diameter / 2) * 1e-3
can_volume = math.pi * (r_outer**2 - r_inner_can**2) * h_total
can_mass = can_density * can_volume

# Jellyroll mass estimated from electrode/separator densities and computed geometry
# (Use reasonable estimate; PyBaMM will compute internal electrochemistry regardless)
```

These map to PyBaMM thermal parameters:
- `"Total heat transfer coefficient [W.m-2.K-1]"` = 5.0 (natural convection, standard assumption)
- `"Cell volume [m3]"` = cell_volume
- `"Cell thermal expansion coefficient [m.m-1.K-1]"` — keep Chen2020 default

**Method: `compute_derived_features(params: dict[str, float]) -> dict[str, float]`**

Computes the 3 derived features (for ML, not for PyBaMM):
```python
{
    "surface_to_volume": 2/r + 2/h,  # using can outer radius and cell height in m
    "tab_conductance_proxy": n_tabs * tab_width,
    "diffusion_path_proxy": electrode_thickness / max(porosity, 1e-6),
}
```

**Method: `get_design_space() -> dict[str, tuple[float, float]]`**

Returns the parameter ranges as a dict (for LHS sampling in P2).

### File 2: `forge/ml/simulation/pybamm_runner.py`

**Purpose:** Implement the `SimulationRunner` ABC. Runs PyBaMM DFN simulations and extracts rate capability and max temperature.

**Class: `PyBaMMRunner(SimulationRunner)`**

**Constructor: `__init__(self, base_parameter_set: str = "Chen2020")`**

Stores the base parameter set name. Does NOT load PyBaMM at init time (lazy loading).

**Method: `simulate(self, cell_spec: CellSpec, config: SimulationConfig) -> SimulationResult`**

This is the main entry point. For each cell design:

1. Create a `GeometryTranslator` instance
2. Build the parameter override dict from `cell_spec`
3. Load PyBaMM base parameter set (Chen2020)
4. Apply overrides
5. Run 3 experiments (see below)
6. Extract targets
7. Return `SimulationResult`

**The 3 experiments per sample:**

**Experiment 1 — Low-rate discharge (0.2C):**
```python
model = pybamm.lithium_ion.DFN()
param = pybamm.ParameterValues("Chen2020")
param.update(overrides)  # from translator

experiment = pybamm.Experiment([
    "Discharge at 0.2C until 2.5V"
])
sim = pybamm.Simulation(model, parameter_values=param, experiment=experiment)
sol = sim.solve()
Q_02C = sol["Discharge capacity [A.h]"].entries[-1]
```

**Experiment 2 — High-rate discharge (3C):**
```python
# Same model setup, different experiment
experiment = pybamm.Experiment([
    "Discharge at 3C until 2.5V"
])
sim = pybamm.Simulation(model, parameter_values=param, experiment=experiment)
sol = sim.solve()
Q_3C = sol["Discharge capacity [A.h]"].entries[-1]
```

**Experiment 3 — Fast charge with thermal (2C CC-CV):**
```python
# DFN with lumped thermal
thermal_model = pybamm.lithium_ion.DFN(
    options={"thermal": "lumped"}
)
param_thermal = pybamm.ParameterValues("Chen2020")
param_thermal.update(overrides)
param_thermal.update({
    "Total heat transfer coefficient [W.m-2.K-1]": 5.0,
    "Ambient temperature [K]": 298.15,  # 25°C
    "Initial temperature [K]": 298.15,
})

experiment = pybamm.Experiment([
    "Charge at 2C until 4.2V",
    "Hold at 4.2V until C/20",
])
sim = pybamm.Simulation(thermal_model, parameter_values=param_thermal, experiment=experiment)
sol = sim.solve()
T_max_K = sol["Cell temperature [K]"].entries.max()
T_max_C = T_max_K - 273.15
```

**Target extraction:**
```python
rate_capability = Q_3C / max(Q_02C, 1e-10)  # prevent division by zero
max_temp = T_max_C
```

**Error handling:**

Some parameter combinations will cause PyBaMM's solver to diverge (e.g., very thick electrode + very low porosity + high C-rate). These must be caught gracefully:

```python
try:
    sol = sim.solve()
except pybamm.SolverError as e:
    return SimulationResult(
        success=False,
        error_message=str(e),
        # ... other fields as None/NaN
    )
```

Log failures with the parameter values that caused them (for debugging and thesis discussion).

**Method: `compute_rate_capability(self, Q_low: float, Q_high: float) -> float`**
```python
return Q_high / max(Q_low, 1e-10)
```

## Important Notes

- The `GeometryTranslator` is the bridge between FORGE's parametric design space and PyBaMM's electrochemical model. This is a thesis-relevant contribution — "we bridge parametric cell geometry to electrochemical simulation."
- The Chen2020 parameter set assumes NMC532/Graphite chemistry. The geometric overrides change cell dimensions while keeping the electrochemistry fixed. This is the standard approach in the literature (Jackowska et al. 2025 used PyBOP the same way).
- Contact resistance as a proxy for tab geometry in 1D DFN is a standard lumped approximation. It's not perfect (no spatial current distribution), but it's defensible for a 1D model. The thesis should acknowledge this limitation.
- Some parameter combinations WILL cause solver failures. This is expected and correct — not all geometric designs are electrochemically viable. The pilot script (P3) will handle this by logging and skipping failed samples.
