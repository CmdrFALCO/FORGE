# Run 002 — Autoresearch on PyBaMM DFN Dataset

## Context

Run 001 ran 200 experiments on synthetic proxy data, achieving 57.5% improvement (0.3351 → 0.1517). Run 002 uses the same autoresearch framework but on a **real physics dataset** generated by PyBaMM DFN simulations (496 samples, Chen2020 parameter set, 4680-class cylindrical cells).

This is the thesis-critical run. The results from this run demonstrate whether the autoresearch methodology works on real simulation data, not just hand-crafted formulas.

## Setup Steps

```bash
cd /path/to/FORGE  # adjust to actual repo path
```

### Step 2: Reset surrogate.py to baseline

**Read the current `experiments/ml/autoresearch/surrogate.py`** to understand its structure. Then reset it to a clean baseline:

- **Architecture:** 2-layer MLP, input → 64 → 64 → 2 outputs
- **Activation:** ReLU
- **Optimizer:** Adam, lr=1e-3
- **Loss:** MSE
- **Batch size:** 64
- **No feature engineering** — use the 11 raw input features only (8 geometric + 3 derived)
- **No BatchNorm, no skip connections, no ensemble** — single model
- **ENSEMBLE_SIZE = 1** (baseline, agent can increase later)
- **Standard normalization** of inputs (zero mean, unit variance from training set)

Keep the output format identical (the 10 required metric keys after the `---` marker). Keep the `--dataset`, `--seed`, `--budget-seconds` CLI arguments. Keep the data loading logic (reads .npz + metadata.json, verifies hashes).

**IMPORTANT:** The baseline must be as simple as possible. Every improvement the agent makes from this starting point is a measurable, attributable gain. Do NOT include any of Run 001's discoveries (no log transforms, no Bruggeman, no BatchNorm, no skip connections).

## Launch

After setup is verified, the autoresearch agent can be launched following the instructions in `program.md` (Option A — agent runs surrogate.py directly, controls accept/reject via git commit/revert).

The budget per experiment should be 300 seconds (same as Run 001). Target: 200 experiments.

**Expected total runtime:** ~18-20 hours (200 × ~5 min average, including overhead).

## What to Monitor

- **Early experiments (1-20):** Should see architecture improvements (activation functions, hidden size, normalization). Similar trajectory to Run 001.
- **Mid experiments (20-100):** Feature engineering phase. This is where it gets interesting — will the agent discover the same physics-informed features (Bruggeman, log transforms) when working with real DFN data instead of formulas that explicitly encode those relationships?
- **Late experiments (100-200):** Hyperparameter tuning and diminishing returns.
- **Key question:** Does the agent discover that n_tabs and tab_width are dead features? If it tries log(tab_conductance) or tab interaction features and they all get rejected, that independently confirms the Sobol screening findings.

## Success Criteria

- Score improves from baseline by at least 30% (lower bar than Run 001's 57.5% because real physics data is harder than synthetic)
- Agent discovers at least 2 physics-informed features that get accepted
- No crashes or data corruption over 200 experiments
- Results are reproducible (deterministic seed, committed surrogate.py after each accepted experiment)
