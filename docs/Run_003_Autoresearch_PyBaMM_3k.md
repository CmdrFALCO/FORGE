# Run 003 — Autoresearch on 3000-Sample PyBaMM DFN Dataset

## Context

Run 002 converged at experiment 37 with 28.4% improvement on 496 PyBaMM DFN samples (347 training). The agent couldn't use ensemble methods, BatchNorm, or deeper architectures because all caused overfitting on the small training set. Run 003 removes that bottleneck with a 3000-sample dataset (~2100 training samples).

**Dependencies:**
- The run archive prompt must be completed first (runs/ directory structure exists)
- Execute on `main` branch, then create a feature branch for the autoresearch loop

## What to Expect

With ~2100 training samples (6× more than Run 002's 347), the agent should be able to:

- **Unlock ensemble methods** — Run 001 used ensemble=30 as its final architecture. Run 002 rejected ensemble because 347 samples couldn't support it. With 2100 samples, ensemble should be viable again.
- **Unlock BatchNorm and skip connections** — Both were rejected in Run 002 for overfitting. More data should make regularization techniques net-positive.
- **Push past 30% improvement** — Run 002 hit 28.4% and converged at exp 37. With more data, the optimization landscape opens up.
- **Confirm the same physics findings** — Bruggeman tortuosity, electrode-porosity interaction, and dead tab features should be rediscovered. If they are, that's three-run consistency.
